Role-gating: how should restricted settings appear?

This shapes the entire settings screen for Friends & Family and Premium users

A

Hidden rows (Recommended)

Admin-only rows simply don't appear for lower-tier users. The settings screen is clean and uncluttered. Users never see features they can't use. This matches how Netflix/Apple TV+ handle profile permissions.

Pros

  • Cleanest UX
  • No confusion
  • Matches your description

Cons

  • No visibility that admin features exist
B

Locked/greyed rows

Admin-only rows are visible but shown with a lock icon and 50% opacity. Tapping shows "Only the admin can change this." More transparent — users know the feature exists.

Pros

  • Users understand why things are missing
  • Good if users might want to request access

Cons

  • Cluttered for F&F users
  • Confusing on a shared device
C

Completely separate screens per role

Each role renders a different settings screen (AdminSettingsScreen, FamilySettingsScreen, etc). Maximum control over the experience per role.

Pros

  • Perfect tailoring per role

Cons

  • 3–4x the code to maintain
  • Easy to diverge and get stale
