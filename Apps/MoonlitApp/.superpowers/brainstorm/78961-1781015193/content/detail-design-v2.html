How it works — TMDB Person API
Step 1 — Get the TMDB person ID from cast data
1
The meta detail response from addons includes a cast array. Each cast member has a name and sometimes a TMDB person ID (tmdb_id).
2
If no ID: call GET /search/person?query=Harold+Perrineau with your TMDB API key. First result is the match. Cache the resolved ID locally.
Step 2 — Fetch bio + credits
1
Call GET /person/{id}?append_to_response=combined_credits — single request returns bio, personal info, and full filmography in one shot.
2
combined_credits.cast contains every title (movies + series) sorted by release date. Group by year, show episode count for series, character name, TMDB vote average.
3
Profile photo from profile_path → https://image.tmdb.org/t/p/w185{profile_path}. Single image, no strip needed.
Caching strategy
Person data is fetched on demand (tap cast card) and cached in memory for the session. No pre-fetching. TMDB key comes from the user's existing Metadata Integration settings — no new key needed.
