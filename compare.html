Design verdict: React + Vite wins
Every design component you'd want for Luna exists as a polished library. Framer Motion gives you Netflix-style transitions. Radix + shadcn gives you accessible dialogs, sliders, and menus out of the box. Vidstack is a production-ready video player with custom skin support. You're not building from scratch — you're assembling.

Design verdict: Faster app, more DIY UI work
Svelte's built-in animation primitives are genuinely elegant — tweened values, spring physics, and transitions are first-class. But the component ecosystem is thin. For Luna specifically, you'd be writing your own video player controls, carousels, and context menus from scratch. The UI will look exactly as good as you make it — there's just no shortcut library ecosystem to lean on.

Design verdict: Near-zero overhead, near-zero component help
Solid.js is blazing fast and the JSX feels familiar, but the design ecosystem barely exists yet. For Luna you'd be building almost every UI component — carousels, player controls, profile cards, modals — from scratch. The result can be pixel-perfect, but it's a solo developer's nightmare if you want to ship quickly. Best suited for someone who loves building UI primitives.

Design verdict: Same as React, just heavier
Next.js uses React under the hood, so you get the exact same component ecosystem — shadcn, Radix, Framer Motion, Vidstack, all of it. The only design-related downside is that Next.js adds page-load overhead that you feel on route transitions, and the App Router's streaming model can cause layout flashes if you're not careful with loading states. Design-wise you lose nothing vs React + Vite.
