Where should the admin panel live?

This affects how you deploy, authenticate, and share code between Luna and its admin panel

A

Separate repo + separate Vercel site

A brand new project at luna-admin.vercel.app. Completely isolated — its own codebase, own deploy pipeline. Uses the same Supabase project for data.

Pros

  • Fully isolated — a bug in admin can't break Luna
  • Can use a different stack (e.g. shadcn/ui dashboard template)
  • Separate deploy permissions

Cons

  • Two repos to maintain
  • Can't share types/utils without a package
B

Same repo, separate Vercel project (Recommended)

Add an /admin Next.js app inside the existing LunaWeb repo. Deployed as a second Vercel project pointing at the same GitHub repo. Shares Supabase client and types.

Pros

  • Shared types, Supabase client, and API helpers
  • One repo to manage
  • Faster to build — reuse existing code

Cons

  • Slightly more complex Vercel setup
  • Shared deploy pipeline
C

Same repo, subdirectory route (/admin/*)

Keep everything in one Next.js app. Admin pages live at luna-web.vercel.app/admin/... — same site, role-protected routes.

Pros

  • Simplest setup — no extra deploy
  • Already have /admin/page.tsx started

Cons

  • Admin and app share the same URL/deploy
  • Less isolation
